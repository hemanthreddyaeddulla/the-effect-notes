Ep 17Closing Causal Pathways, and Collider Variables

▶ Watch the video 13:40
On this pageThe 30-second versionWhy this existsThe ideasTrapsSay it out loudRecall drillConnectionsGlossary

A path is open when every variable on it varies and closed when even one doesn't — so you close bad paths by removing variation, either by controlling for a variable or by picking a sample that doesn't vary in it; except that colliders invert the rule, sitting on paths that are already closed until you control for them and blow them open.

The 30-second version

  • Open path: every variable along it is allowed to vary in your sample. It can induce a relationship.
  • Closed path: at least one variable on it has no variation. It can't do anything.
  • Two ways to close: condition/control for a variable, or select a sample with no variation in it.
  • Kill one variable and every path it sits on closes at once.
  • Don't close good paths. Controlling for something your treatment caused is post-treatment bias.
  • A collider is a variable on a path with both arrows pointing at it. That path is pre-closed — free of charge.
  • Control for a collider and the path opens. You can do this by accident, just by choosing a sample.
  • Collider is a property of a variable on a particular path, not a global label.

Why this exists

You've got your paths sorted into good and bad. Identification means closing the bad ones — but nothing so far says what "closing" physically is or how to do it.

And there's a trap waiting. The instinct that follows from "close bad paths" is control for everything you can. Two things on this page make that instinct actively dangerous.

The ideas

1

Open and closed

Plain English. A path works only if everything along it is free to vary. Freeze any one variable and the whole path stops working.

Precisely. A path is open when all variables on it are allowed to vary in your sample. It's closed when at least one variable on it has no variation. A closed path can no longer induce a relationship between the variables at its ends.

Anchor image. A row of dominoes. Every one has to be free to fall. Glue any single domino down and nothing past it moves — the whole line is dead, no matter how many are still standing.

Worked example. Wine ← Health → Lifespan. In ordinary data there are people who drink more and less, people healthier and less healthy, people who live longer and shorter. Variation everywhere, so the path is open — it's a live reason wine and lifespan correlate.

Now suppose you either statistically adjust for health, or pick a sample where everyone is naturally healthy. Either way there's no variation in health left. You can no longer say "naturally healthy people drink more and live longer" — in your data there are no more and less healthy people. The path is closed.

Where it breaks — and this is the useful part: killing health's variation closes every path health sits on, all at once. Wine ← Income ← U1 → Health → Lifespan shuts down too.

2

The two ways to close a path

Plain English. Adjust it away statistically, or choose data that doesn't have it.

Precisely.

  1. Conditioning / controlling. Measure the variable and adjust for it, making the data behave as though it doesn't vary. Take out the relationship between income and wine, and between income and lifespan, and income no longer influences either.
  2. Sample selection. Choose a dataset with no variation in it — only healthy people, say.

Where it breaks. Both work. Only the first is deliberate every time; the second can happen to you without your noticing, which is the collider trap below.

3

Worked all the way through: wine and lifespan

Plain English. Six paths. Two you want, four you don't. Two control variables close all four.

Worked example. Walk out every path from wine to lifespan and you get six.

Front doors (good) — leave open:

  1. Wine → Lifespan
  2. Wine → Drugs → Lifespan

Back doors (bad) — must close:

  1. Wine ← Income → Lifespan
  2. Wine ← Income ← U1 → Health → Lifespan
  3. Wine ← Health → Lifespan
  4. Wine ← Health ← U1 → Income → Lifespan

Control for income. Income sits on paths 3, 4, and 6 — all three close at once. What's left open: the two front doors, plus path 5.

Control for health as well. Path 5 has only health on it, so health is the only thing that could close it — there's nothing else to control for. Now paths 3–6 are all shut.

What remains: exactly the two front doors. The effect is identified.

4

Don't control for everything

Plain English. Drugs is obviously related to all this. Control for it and you destroy part of your answer.

Precisely. Controlling for drugs would close Wine → Drugs → Lifespan — a front door you wanted open. You'd be removing part of the real effect. Controlling for something that is affected by your treatment is post-treatment bias.

Anchor image. Measuring how much a tap fills a bucket while clamping the pipe halfway along. The tap works fine; you disabled the route you were trying to measure.

Where it breaks. The instinct "it's related, so I should control for it" is exactly wrong here. There are things you must not control for, and the diagram is what tells you which.

5

Colliders

Plain English. A variable that two things point into. Any path through it is already closed — until you control for it, at which point it opens.

Precisely. A collider is a variable on a path where both arrows on that path point toward it. A path containing a collider is pre-closed — it can't induce a relationship, and you don't have to do anything about it. But controlling for the collider opens it back up, and it becomes a problem.

Critically: being a collider is a property of a variable on a particular path, not a fixed label. The same variable can be a collider on one path and not on another.

Anchor image. A door that's locked by default. Leave it alone and nobody comes through. The only way to create a problem is to walk over and unlock it.

Worked example. Does tutoring in high school raise intelligence? Say you even randomized the tutoring, so you'd think you're fine.

Tutoring helps you get into college regardless of intelligence. Intelligence also helps you get into college. So Tutoring → College ← Intelligence is a path — and it's pre-closed, because college is a collider on it. No problem.

Now: you decide to survey college students, because they're easy to survey.

You have just controlled for college. The path opens. And look at who is in that sample:

  • Smart people who got in without needing tutoring
  • People who got in because they were tutored, but weren't especially smart
  • People who were both

Within college students, being tutored and being smart are substitutes for getting in. So tutoring and intelligence will look negatively related in your data — even if they weren't related at all to begin with.

Where it breaks. Nobody chose to control for college. It happened through a sampling decision that felt like convenience. This is the everyday version of the collider trap.

Say it out loud

So you've got bad paths to close. What does closing one actually mean?

A path is open when every variable on it is free to vary. It's closed when even one variable on it doesn't vary at all. Think dominoes — glue down any single one and nothing past it moves.

So: wine ← health → lifespan. In normal data, health varies, so that path is live — healthy people drink more and live longer, and that shows up as wine correlating with lifespan. Now suppose I control for health, or I only sample people who are naturally healthy. Either way there's no variation in health left. I can't tell that story anymore. Path's dead.

And here's the efficient bit: killing one variable closes every path it sits on.

Now the part that trips people up. The obvious lesson seems to be "control for everything." It's wrong, twice over.

First: in my wine diagram, wine affects your drug taking, which affects your lifespan. That's a real effect of wine and I want to count it. If I control for drugs, I close it — I've deleted part of my own answer. That's post-treatment bias: controlling for something your treatment caused.

Second, and this one's beautiful. A collider is a variable where two arrows point into it. And a path through a collider is already closed — free of charge, nothing to do. But if you control for it, it opens.

Watch how easily that happens. Does high-school tutoring raise intelligence? Tutoring helps you get into college. So does being smart. Fine — college is a collider, path's pre-closed. Now I go survey some students... at a college. Because that's convenient.

I just controlled for college. And now look who's in my sample: smart kids who didn't need tutoring, and tutored kids who weren't especially smart. In that group, tutoring and intelligence look negatively related — and there might be no relationship at all.

I never typed a control variable. I just picked a sample.

Recall drill

QWhen is a path open, and when is it closed?

Open when every variable on it is allowed to vary in your sample. Closed when at least one variable on it has no variation.

QWhat are the two ways to close a path?

Condition or control for a variable on it, or select a sample in which that variable has no variation.

QYou control for health. Which paths close?

Every path that health sits on, all at once — not just the one you had in mind.

QIn the wine diagram, controlling for income closes which paths, and what's left?

Paths 3, 4, and 6 — every back door income sits on. Left open: the two front doors plus path 5, Wine ← Health → Lifespan.

QWhy must you control for health specifically to close path 5?

Health is the only variable on that path, so there's nothing else you could control for to shut it.

QWhy should you not control for drugs?

Drugs is on the front door path Wine → Drugs → Lifespan, which you want counted. Closing it removes part of the real effect.

QWhat is post-treatment bias?

Controlling for a variable that is affected by your treatment, which closes a front door path and deletes part of the effect you're trying to measure.

QDefine a collider.

A variable on a path where both arrows on that path point toward it.

QWhat's the default status of a path containing a collider, and what changes it?

It's pre-closed — no action needed. Controlling for the collider opens it.

QIs "collider" a property of a variable or of a variable on a path?

Of a variable on a particular path. The same variable can be a collider on one path and not on another.

QTutoring → College ← Intelligence. You survey only college students. What happens and why?

You've controlled for college, opening the path. Within college students, tutoring and being smart are substitute routes in, so they appear negatively related — even if unrelated in the population.

QYou randomized your treatment. Are you safe from collider bias?

No. Tutoring was randomized and the college-student sample still produces a spurious negative relationship, entirely from sample selection.

Glossary

Open path
a path on which every variable is free to vary, so it can induce a relationship.
Closed path
a path on which at least one variable has no variation, so it cannot.
Conditioning
adjusting for a variable so the data behaves as though it doesn't vary.
Sample selection
choosing data with no variation in a variable, which closes paths just as controlling does.
Post-treatment bias
controlling for a variable affected by your treatment, closing a front door path you wanted open.
Collider
a variable on a path with both arrows pointing at it; the path is pre-closed unless you control for the collider.
